# Demo Video Script

Target length: 3 minutes

## 0:00-0:15 Opening

Show the live FadFada app at `https://fad-fada.vercel.app`.

Voiceover:

"This is FadFada, a bilingual AI wellbeing companion for people who need a private space to talk freely, feel understood, and move forward. It works in English and Arabic, it avoids mixed-language screens, and it is built as a Progressive Web App for phones, laptops, tablets, and shared computers."

Shot checklist:

- hero section
- FadFada | فضفضة brand
- language switcher
- PWA/install area if visible

## 0:15-0:35 Bilingual And Persona Experience

Switch language from English to Arabic, then back if needed. Select a persona.

Voiceover:

"FadFada is not a generic chatbot. Users can choose a conversation character with a clear identity, and the interface supports right-to-left Arabic and left-to-right English from the first version."

Shot checklist:

- Arabic UI direction
- persona strip
- selected persona portrait/name

## 0:35-1:05 Support Conversation

Use the in-app Demo Runbook and tap the pressure/support scenario.

Voiceover:

"The core experience is a professional full-screen chat. The user can write naturally, and FadFada responds in the same language with a warm reflection and a small next-step plan. The goal is not to diagnose. The goal is to help the user feel heard first, then move toward one practical action."

Shot checklist:

- full-screen chat opening
- sender and companion bubbles
- adaptive presence orb
- next-step plan panel

## 1:05-1:35 Learning Room

Tap the Learning Sprint scenario from the Demo Runbook and open/maximize a resource if one appears.

Voiceover:

"When the user asks for learning support, FadFada changes mode and opens a Learning Room inside the conversation. It can recommend videos, articles, courses, and document-style resources without breaking the user's flow."

Shot checklist:

- Learning Room card inside chat
- resource chooser
- maximized viewer
- Word-like document pages or video thumbnail

## 1:35-2:00 Voice And Video Modes

Show text/voice/video mode selector and recording controls. Do not record private content.

Voiceover:

"FadFada is designed for text, voice, and video reflection. The current prototype includes microphone and camera permission flows, recording timers, cancel actions, browser voice playback, and a shared reflection pipeline for recorded notes."

Shot checklist:

- mode selector
- microphone flow
- video preview or camera permission state
- timer/cancel/analyze buttons

## 2:00-2:25 Safety Flow

Tap the Safety Check scenario from the Demo Runbook.

Voiceover:

"Because this is a wellbeing product, safety is a product requirement. If high-risk language appears, FadFada interrupts the normal companion flow, avoids therapy or diagnosis claims, and routes the user toward urgent human support and crisis resources."

Shot checklist:

- warning-style assistant bubble
- crisis plan
- Find a Helpline CTA

## 2:25-2:50 Evidence Room

Scroll to Evidence Room and click export evidence.

Voiceover:

"For the Build with Gemini XPRIZE, we also built an Evidence Room. It shows PWA readiness, live Vertex AI readiness, deployment version, business positioning, and proof signals. The team can export a JSON evidence file for judges and Devpost review."

Shot checklist:

- Evidence Room metrics
- proof signals
- Vertex AI readiness line
- Export evidence button
- downloaded JSON file name

## 2:50-3:00 Close

Show final product screen or README/Devpost title.

Voiceover:

"FadFada is built to become a real bilingual AI-operated wellbeing business: start with a founding beta, collect evidence, improve safety, and help people feel heard before they are ready to ask for more formal support."

Shot checklist:

- product hero or focused chat
- founding beta message
- final brand frame

## Recording Notes

- Keep private messages fictional and demo-safe.
- Do not claim therapy, diagnosis, treatment, or emergency support.
- Show both English and Arabic briefly, but keep narration in English for international judges.
- Use the Judge Demo Path buttons to avoid typing during recording.
- Export evidence near the end so judges see the artifact.
